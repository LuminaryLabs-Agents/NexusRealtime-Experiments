# Domain Scope

This file lists the possible domains touched by the idea.

A domain is the broad scope of meaning that a service kit acts on.

Examples:

```txt
movement
combat
resource pressure
camera
lighting
audio
terrain
agent behavior
scoring
progression
input
time pressure
```

## Domain boundary rules

Each domain should own one reusable area of meaning.

Do not make domains based on the game title.

Good:

```txt
resource-pressure-domain
route-checkpoint-domain
camera-cinematic-domain
hazard-director-domain
```

Bad:

```txt
neon-racer-domain
goblin-office-domain
my-game-domain
```

## Domain fit audit

For each proposed domain, answer:

```txt
Is this reusable across multiple games?
Is this a broad system of meaning?
Could this be an existing domain update instead?
Could this be a preset instead of a new domain?
Could this be a one-off bridge in the experiment?
```
