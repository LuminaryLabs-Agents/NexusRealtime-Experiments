# Prototype Plan

## Prototype question

What one question should this prototype answer?

## Smallest playable version

```txt
one arena / scene
one player action
one threat / obstacle
one scoring or progress rule
one failure state
one restart loop
```

## Must-have features

- Start a run.
- Perform the core action.
- Score or progress.
- Escalate pressure.
- Fail or clear.
- Restart quickly.

## Nice-to-have features

- Extra enemy types.
- Extra levels.
- Juice and polish.
- Attract mode.
- Cabinet-specific feedback.

## Input map

```txt
move:
primary action:
secondary action:
pause / home:
restart:
```

## Prototype success criteria

The prototype succeeds if:

- The loop is understandable within 5 seconds.
- The controls feel immediate.
- The player wants one more run.
- The failure state is clear.
- The scoring or progress is readable.

## Prototype failure criteria

The prototype fails if:

- The idea needs too much explanation.
- The controls feel bad.
- The loop is not fun after one minute.
- The screen is hard to read.
- The concept requires too much content before becoming fun.
